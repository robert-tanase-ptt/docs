## Why Notifications?

Notifications keep you informed about important security events without constantly checking the dashboard.

<CardGroup cols={2}>

## Notification Events

| Event | Description |
|-------|-------------|
| Scan Started | Scan has begun |
| Scan Completed | Scan finished successfully |
| Scan Failed | Scan encountered an error |
| New Finding | Vulnerability discovered |
| Critical Finding | High-severity issue found |

## Best Practices

<Tip>
Start with fewer notifications and add more as needed. Too many notifications leads to alert fatigue.
</Tip>

<AccordionGroup>
  <Accordion title="Prioritize Critical Alerts">
    Set up immediate notifications for critical and high severity findings.
  </Accordion>
  <Accordion title="Use Appropriate Channels">
    Email for summaries, Slack for real-time alerts, Jira for tracking.
