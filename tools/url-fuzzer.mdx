## Overview

The URL Fuzzer discovers hidden content on web servers by testing common file and directory names. It finds backup files, admin panels, configuration files, and other resources not linked from the main application.

## Configuration options

### Extensions

Add file extensions to test:
- PHP: `.php`, `.php.bak`
- ASP: `.asp`, `.aspx`
- Backup: `.bak`, `.old`, `.backup`
- Config: `.conf`, `.config`, `.yml`

### Recursion

When enabled, discovered directories are also fuzzed:
- Set maximum depth
- Control directory-only recursion

### Response filtering

Filter results by:
- **Status codes** — Include/exclude HTTP status codes
- **Response size** — Filter by content length
- **Response content** — Match/exclude patterns

<Tip>
Filter out common "not found" response sizes to reduce noise in results.
</Tip>

## Results

For each discovered resource:
- **URL** — Full path to resource
- **Status code** — HTTP response code
- **Size** — Response content length
- **Content type** — MIME type

## Authentication

The URL Fuzzer supports custom headers in **Custom** scan mode, allowing you to fuzz authenticated endpoints.

**How to configure:**
1. Select **Custom** scan type
2. Navigate to **Request options → Headers**
3. Enter your authentication headers

**Use for:**
- JWT tokens
- Authorization headers
- Cookies
- Custom User-Agent strings

**Example:**
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
Cookie: session_id=abc123
User-Agent: Custom-Scanner/1.0
```

[Full authentication guide →](/using/scans/authenticated-scanning#url-fuzzer)

## Security implications

Discovering hidden content can reveal:
- **Backup files** with source code or credentials
- **Configuration files** exposing database passwords
- **Admin interfaces** for privilege escalation
- **Version control** directories with full source
- **Debug endpoints** with sensitive information

<Warning>
Exposed .git directories can leak entire source code repositories including commit history.
</Warning>
