## Overview

Sniper is an automated penetration testing tool that combines reconnaissance, scanning, and exploitation capabilities. It streamlines the testing workflow by automating common attack patterns.

- **Vulnerability type** — Category and name
- **Target** — Affected host/service
- **Evidence** — Proof of exploitation
- **Severity** — Risk rating
- **Details** — Technical description

</Warning>

## Authenticated scanning

Sniper supports authenticated scans via four protocols, allowing you to extract system artifacts through an authenticated session.

<Note>
Authenticated Sniper scans don't search for vulnerabilities—instead, they extract artifacts (system information, configurations, user data) through the authenticated session. This is useful for post-authentication reconnaissance.
</Note>

| Protocol | Authentication | Use case |
|----------|----------------|----------|
| **SSH** | Username/password or private key | Linux/Unix systems |
| **WinRM** | Username/password (NTLM or Basic) | Windows systems |
| **SMB** | Username/password with optional domain | Windows file shares |
| **MySQL** | Username/password with database name | Database servers |

### SSH authentication

**Username/Password:**
1. Select **SSH** protocol
2. Choose **Username and Password** authentication
3. Enter SSH credentials

**SSH Private Key:**
1. Select **SSH** protocol
2. Choose **SSH private key** authentication
3. Paste your private key content
4. Enter the key passphrase (if applicable)

### WinRM authentication

1. Select **WinRM** protocol
2. Enter **username** and **password**
3. Choose authentication type: **NTLM** (domain) or **Basic** (standalone)

### SMB authentication

1. Select **SMB** protocol
2. Enter **username** and **password**
3. Optionally specify the **domain** for domain-joined systems

### MySQL authentication

1. Select **MySQL** protocol
2. Enter **username** and **password**
3. Specify the **database name** to connect to
