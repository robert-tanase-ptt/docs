## Overview

Website Recon performs reconnaissance on web applications, identifying technologies, frameworks, and configuration details to support security assessment planning.

## Information gathered

### Technology detection

| Category | Examples |
|----------|----------|
| **CMS** | WordPress, Drupal, Joomla |
| **Frameworks** | React, Angular, Vue, Laravel, Django |
| **Web servers** | nginx, Apache, IIS, LiteSpeed |
| **Languages** | PHP, Python, Java, Node.js |
| **JavaScript** | jQuery, Bootstrap, Lodash |
| **CDN** | Cloudflare, Akamai, Fastly |

### Security analysis

- HTTP security headers present/missing
- Cookie attributes (HttpOnly, Secure, SameSite)
- SSL/TLS configuration overview
- Content Security Policy presence
- X-Frame-Options status
- HSTS implementation

### Meta information

- Page titles and descriptions
- robots.txt content and directives
- sitemap.xml locations
- Interesting paths discovered
- Contact information

## Why recon matters

Technology fingerprinting helps:
- **Select appropriate scanners** — Use CMS-specific tools when detected
- **Identify version-specific vulnerabilities** — Match versions to CVEs
- **Understand application architecture** — Plan targeted testing
- **Prioritize testing areas** — Focus on known-vulnerable components

<Tip>
Use Website Recon results to choose which specialized scanners to run. For example, if WordPress is detected, follow up with the [WordPress Scanner](/tools/wordpress-scanner).
</Tip>

## Following up

Based on recon results, consider these next steps:

| Discovery | Recommended action |
|-----------|-------------------|
| WordPress | Run [WordPress Scanner](/tools/wordpress-scanner) |
| Drupal | Run [Drupal Scanner](/tools/drupal-scanner) |
| Joomla | Run [Joomla Scanner](/tools/joomla-scanner) |
| API endpoints | Run [API Scanner](/tools/api-scanner) |
| Outdated software | Check CVE databases |
| Missing headers | Document for remediation |
