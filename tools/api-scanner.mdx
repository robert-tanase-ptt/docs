## Overview

The API Scanner tests REST and GraphQL APIs for security vulnerabilities. It understands API specifications (OpenAPI/Swagger, Postman) to perform comprehensive security testing of your API endpoints.

  <Card title="Authentication issues" icon="key">

## API specification support

### OpenAPI/Swagger

Provide your API specification to enable comprehensive testing:
- **URL**: Link to hosted OpenAPI/Swagger JSON or YAML
- **File upload**: Upload specification file directly

### Postman collections

Import Postman collections for testing:
- **URL**: Link to shared Postman collection
- **File upload**: Upload exported collection JSON

### GraphQL

For GraphQL APIs, the scanner uses introspection to discover:
- Available queries and mutations
- Input types and arguments
- Schema structure

<Note>
Providing an API specification dramatically improves test coverage by ensuring all endpoints are tested.
</Note>

## Vulnerability categories

Tests aligned with OWASP API Security Top 10:

| Category | Description |
|----------|-------------|
| **Broken Object Level Authorization** | Accessing other users' data |
| **Broken Authentication** | Weak authentication mechanisms |
| **Excessive Data Exposure** | APIs returning too much data |
| **Lack of Resources & Rate Limiting** | Missing rate limits |
| **Broken Function Level Authorization** | Unauthorized function access |
| **Mass Assignment** | Binding client data to internal objects |
| **Security Misconfiguration** | Insecure default configurations |
| **Injection** | SQL, NoSQL, command injection |

## Authentication

Configure authentication for testing protected API endpoints using custom headers.

| Method | Description |
|--------|-------------|
| **Headers** | Add Authorization headers, API keys, Bearer tokens |

**How to configure:**
1. In the scan configuration, locate the **Headers** section
2. Enter your authentication headers (one per line)

**Format:**
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
X-API-Key: your-api-key-here
```

**Common header examples:**

| Use case | Header format |
|----------|---------------|
| JWT Bearer token | `Authorization: Bearer <token>` |
| Basic authentication | `Authorization: Basic <base64-credentials>` |
| API key | `X-API-Key: <key>` |
| Custom token | `X-Auth-Token: <token>` |

<Tip>
If your API uses multiple authentication mechanisms (e.g., both a Bearer token and an API key), include all required headers.
</Tip>
