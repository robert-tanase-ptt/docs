## Overview

The Subdomain Finder discovers subdomains of a target domain, revealing additional attack surface that may host vulnerable applications, development servers, or forgotten assets.

## Enumeration methods

| Method | Description |
|--------|-------------|
| **DNS brute force** | Test common subdomain names |
| **Certificate transparency** | Query CT logs for issued certificates |
| **DNS zone transfer** | Attempt AXFR if misconfigured |
| **Public sources** | Search engines, archives, databases |
| **Passive DNS** | Historical DNS record databases |

## Results

For each subdomain:
- **Subdomain name** — Full hostname
- **IP addresses** — Resolved A/AAAA records
- **HTTP status** — If web server responds
- **Title** — Web page title if available

## Why subdomain enumeration matters

Subdomains often host:
- **Development environments** — Less security, more bugs
- **Legacy applications** — Outdated and vulnerable
- **Internal tools** — Admin panels, dashboards
- **Forgotten assets** — Abandoned but still running
- **Third-party services** — Potentially misconfigured

<Tip>
After discovering subdomains, check for takeover risks with the [Subdomain Takeover](/tools/subdomain-takeover) scanner.
</Tip>
