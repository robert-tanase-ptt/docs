## Overview

The WordPress Scanner identifies vulnerabilities in WordPress installations, including core, plugins, and themes. It uses WPScan technology to detect security issues specific to the WordPress ecosystem.

## Scan profiles

| Profile | Coverage | Use case |
|---------|----------|----------|
| **Light** | Basic enumeration and checks | Quick assessment |
| **Deep** | Full enumeration and vulnerability detection | Comprehensive audit |

## Checks performed

### Core checks

| Check | Description |
|-------|-------------|
| **Version detection** | Identify WordPress version |
| **Core vulnerabilities** | Check for known WordPress CVEs |
| **readme.html** | Detect exposed readme file |
| **Debug mode** | Check for WP_DEBUG exposure |
| **Directory listing** | Check uploads and other directories |

### Plugin checks

| Check | Description |
|-------|-------------|
| **Plugin enumeration** | Discover installed plugins |
| **Version detection** | Identify plugin versions |
| **Vulnerability matching** | Check against WPScan database |
| **Abandoned plugins** | Detect unmaintained plugins |

### Theme checks

| Check | Description |
|-------|-------------|
| **Theme detection** | Identify active theme |
| **Version detection** | Determine theme version |
| **Vulnerability matching** | Check for known theme vulnerabilities |

### Configuration checks

| Check | Description |
|-------|-------------|
| **User enumeration** | Discover WordPress users |
| **XML-RPC** | Check for enabled XML-RPC |
| **REST API** | User enumeration via REST API |
| **Backup files** | Check for exposed backups |
| **wp-config.php** | Check for exposed configuration |

## Common findings

| Finding | Risk | Recommendation |
|---------|------|----------------|
| Outdated WordPress | High | Update to latest version |
| Vulnerable plugin | Varies | Update or remove plugin |
| User enumeration | Medium | Disable username discovery |
| XML-RPC enabled | Medium | Disable if not needed |
| Debug mode on | Medium | Disable WP_DEBUG in production |
| Exposed readme.html | Low | Remove or restrict access |

<Tip>
Keep WordPress, plugins, and themes updated. Most WordPress vulnerabilities affect outdated installations.
</Tip>

## Authentication

The WordPress Scanner supports HTTP Basic Authentication in **Custom** scan mode for testing sites protected by server-level authentication.

**How to configure:**
1. Select **Custom** scan type
2. Enable **Basic Authentication**
3. Enter your **username** and **password**

<Note>
Basic Authentication is different from WordPress login credentials. It's typically configured at the server level (Apache/Nginx) to restrict access to the entire site or wp-admin area.
</Note>

## Results

For each finding:
- **Component** — Core, plugin, or theme name
- **Version** — Detected version
- **Vulnerability** — CVE and description
- **Severity** — Risk rating
- **References** — WPScan database links
