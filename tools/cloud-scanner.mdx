## Overview

The Cloud Scanner identifies misconfigured cloud storage buckets and services that may expose sensitive data. It tests for public access on AWS S3, Azure Blob Storage, and Google Cloud Storage.

## Cloud storage checks

### AWS S3

| Check | Risk |
|-------|------|
| Public read access | Data exposure |
| Public write access | Data tampering |
| Public listing | Information disclosure |
| ACL misconfigurations | Unauthorized access |

### Azure Blob Storage

| Check | Risk |
|-------|------|
| Anonymous read access | Data exposure |
| Container listing | Information disclosure |
| Public blob access | Individual file exposure |

### Google Cloud Storage

| Check | Risk |
|-------|------|
| Public object access | Data exposure |
| allUsers permissions | Universal access |
| allAuthenticatedUsers | Any Google account access |

## Common findings

| Finding | Impact | Remediation |
|---------|--------|-------------|
| Public bucket listing | Info disclosure | Disable public listing |
| Public read access | Data breach | Restrict to authorized users |
| Public write access | Data tampering | Remove write permissions |
| Sensitive files exposed | Data breach | Review and restrict access |

<Warning>
Publicly writable buckets allow attackers to upload malicious content or modify existing files.
</Warning>

## Results

For each finding:
- **Resource** — Bucket/container name
- **Provider** — AWS, Azure, or GCP
- **Access level** — Public read, write, list
- **Risk** — Severity rating
- **Details** — Specific misconfigurations

## Best practices

<Tip>
Enable cloud provider security tools like AWS Config Rules, Azure Security Center, or GCP Security Command Center for continuous monitoring.
</Tip>

- **Block public access** by default
- **Use IAM policies** for access control
- **Enable logging** for audit trails
- **Review regularly** for drift
- **Encrypt data** at rest and in transit
