## Security Testing Terms

<AccordionGroup>
  <Accordion title="Asset">
    A resource (domain, IP, URL, or network range) that you've added to the platform for testing.
  </Accordion>

  <Accordion title="Attack Surface">
    The total sum of vulnerabilities and entry points that an attacker could exploit.
  </Accordion>

  <Accordion title="CVE (Common Vulnerabilities and Exposures)">
    A standardized identifier for publicly known security vulnerabilities.
  </Accordion>

  <Accordion title="CWE (Common Weakness Enumeration)">
    A categorization system for software security weaknesses.
  </Accordion>

  <Accordion title="CVSS (Common Vulnerability Scoring System)">
    A standard for rating the severity of security vulnerabilities.
  </Accordion>

  <Accordion title="Finding">
    A security issue or vulnerability discovered during a scan.
  </Accordion>

  <Accordion title="Fingerprinting">
    The process of identifying technologies, software versions, and configurations on a target.
  </Accordion>

  <Accordion title="Penetration Testing (Pentest)">
    Authorized simulated attacks to identify security weaknesses.
  </Accordion>

  <Accordion title="Reconnaissance (Recon)">
    The process of gathering information about a target before testing.
  </Accordion>

  <Accordion title="Scan">
    An automated security test run against one or more targets.
  </Accordion>

  <Accordion title="Scope">
    The defined boundaries of what can be tested in a security engagement.
  </Accordion>

  <Accordion title="Target">
    The specific item being scanned in a particular scan job.
  </Accordion>

  <Accordion title="Triage">
    The process of reviewing and prioritizing findings for remediation.
  </Accordion>

  <Accordion title="Vulnerability">
    A weakness that can be exploited to gain unauthorized access or cause harm.
  </Accordion>

  <Accordion title="Workspace">
    An organizational container for grouping related assets, scans, and findings.
  </Accordion>
</AccordionGroup>
