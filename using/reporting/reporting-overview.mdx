## Why Reporting Matters

Reports transform technical scan data into actionable documents for stakeholders, management, and compliance requirements.

## Report Contents

### Standard Sections

| Section | Description |
|---------|-------------|
| Executive Summary | Overview of findings and risk level |
| Scope | What was tested |
| Methodology | How testing was conducted |
| Findings | Detailed vulnerability descriptions |
| Remediation | Recommendations for each finding |
| Appendices | Supporting data and evidence |

### Findings Information

Each finding in a report includes:
- Title and description
- Severity and risk rating
- Affected assets and locations
- Evidence (screenshots, requests/responses)
- Remediation guidance
- References (CVE, CWE, etc.)

## Output Formats

- **PDF**: Professional documents for sharing
- **Word (DOCX)**: Editable for customization
- **HTML**: Web-based viewing
- **JSON/CSV**: Data export for integration

<Tip>
Use PDF for final deliverables and DOCX when you need to make manual edits or additions.
</Tip>
