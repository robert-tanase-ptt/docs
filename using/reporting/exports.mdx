## Export Options

Export your security data for external tools, analysis, or archiving.

## Available Formats

| Format | Use Case |
|--------|----------|
| **PDF** | Professional reports, client deliverables |
| **DOCX** | Editable reports, customization |
| **CSV** | Spreadsheet analysis, data processing |
| **JSON** | API integration, programmatic access |
| **XML** | Legacy system integration |
| **HTML** | Web viewing, email sharing |

## Exporting Findings

### Individual Finding Export

Export a single finding with all details:
- Full description
- Evidence
- Remediation guidance
- References

### Bulk Finding Export

Export multiple findings at once:

<Steps>
  <Step title="Filter Findings">
    Use filters to select the findings you want.
  </Step>
  <Step title="Select Findings">
    Choose individual items or select all.
  </Step>
  <Step title="Choose Format">
    Select your desired export format.
  </Step>
  <Step title="Download">
    Export and save the file.
  </Step>
</Steps>

## CSV Export Fields

When exporting to CSV, you'll get fields like:

```
Title, Severity, Status, Asset, URL, Tool, Date, Description, Remediation
```

<Tip>
CSV exports are ideal for importing into ticketing systems or creating custom dashboards.
</Tip>

## JSON Export Structure

JSON exports include structured data:

```json
{
  "findings": [
    {
      "id": "finding-123",
      "title": "SQL Injection",
      "severity": "high",
      "asset": "example.com",
      "url": "/api/users",
      "evidence": { ... },
      "remediation": "..."
    }
  ]
}
```

## Integration Uses

Common export integration scenarios:

- **SIEM Integration**: Import findings into security information systems
- **Ticketing**: Create tickets in Jira, ServiceNow, etc.
- **Custom Dashboards**: Build reports in BI tools
- **Compliance**: Archive for audit requirements
