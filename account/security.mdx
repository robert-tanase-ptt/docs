## Security Overview

Protecting your account is critical—it has access to sensitive security testing data.

## Two-Factor Authentication (2FA)

<Warning>
We strongly recommend enabling 2FA for all accounts.
</Warning>

### Enabling 2FA

<Steps>
  <Step title="Go to Security Settings">
    Navigate to Settings → Security.
  </Step>
  <Step title="Enable 2FA">
    Click "Enable Two-Factor Authentication".
  </Step>
  <Step title="Scan QR Code">
    Use an authenticator app to scan the code.
  </Step>
  <Step title="Enter Code">
    Enter the code from your app to verify.
  </Step>
  <Step title="Save Recovery Codes">
    Store backup codes securely.
  </Step>
</Steps>

### Supported Authenticator Apps

- Google Authenticator
- Authy
- Microsoft Authenticator
- 1Password
- Any TOTP-compatible app

### Recovery Codes

<Note>
Save your recovery codes in a secure location. They're the only way to access your account if you lose your authenticator.
</Note>

## Password Security

### Password Best Practices

| Practice | Recommendation |
|----------|----------------|
| Length | 12+ characters |
| Complexity | Mix of types |
| Uniqueness | Unique to this service |
| Manager | Use a password manager |

### Changing Your Password

1. Go to Settings → Security
2. Click "Change Password"
3. Enter current password
4. Enter and confirm new password
5. Click Save

## Session Management

### Active Sessions

View and manage active sessions:
- See all logged-in devices
- View login locations
- Revoke suspicious sessions

### Session Security

| Setting | Description |
|---------|-------------|
| Auto-logout | Session timeout period |
| Remember me | Persistent login option |
| Device trust | Trusted device management |

## Login History

Review your login history for suspicious activity:
- Login timestamps
- IP addresses
- Geographic locations
- Device information

<Tip>
Regularly review your login history for unauthorized access attempts.
</Tip>

## API Key Security

- Generate separate keys for different purposes
- Rotate keys periodically
- Revoke unused keys
- Never share keys in code or logs
